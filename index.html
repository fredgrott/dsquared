Features


Auto-Generated TOC

Table of Contents is auto-generated based on the documents headings for the easiest pain-free way to creater web docs.

Fixed Sidebar and Navbar

MSIE compatible fixed siddebar and navbar that tranistions to non-fixed on mobile.

JSTree File Structures

Visual file structure dispalys using JSTree.

API Code Formating-Highlighting

Code friendly highlighting and formating when you have to document things for your firm's api that 3rd parties use.

SetUp

DSquared is self-contained so that you can quickly prototype a one-page html document of an api or app. The fonts, js, and css files are in the assets sub-folder. The prism componenets that define code highlighting formating per language are in the components sub-folder. The same exact demo is the same exact layout so you can hit-the-ground running and create that one page web doucment in minutes following this demo example.

Fixed Sidebar Navbar

The Stickyfill polyfill works with all browsers even MSIE which is why its being used as I know a large portion of you still have those on MSIE browsers to support.

Stickyfill init

Its a simple setup, first initialize it in your init.js after you have imported the css and js files just like this docs example does:

Javascript


                Stickyfill.add($('.sticky'));

Stickyfill css

We want two things with our stickyfill css setup. One, fixed on desktop for the navbar and sidebar. Two, both elements transition to non-fixed on mobile.

CSS


                .sticky {

                    top: 0;
                }
                @media (min-width: 768px) {

                      .sticky {
                        position: -webkit-sticky;
                        position: sticky;
                        top: 0;
                        z-index: 1071; }
                }
                .sticky:before,
                .sticky:after {
                    content: '';
                    display: table;
                }

The sticky before and after are included as you might be doing fixed headers.

Because I wanted the TOC to be placed lower in the column, I add an extra css class besides just sticky:

CSS


                @media (min-width: 768px) {
                  .doc-sidebar{
                    position: -webkit-sticky;
                    position: sticky;
                    top: auto;
                    z-index: 1071;
                  }

                }

And than I add some extra brs to the sidebar:


                  <div class="col-12 col-md-3 col-xl-2  " id="fixed-sidebar">

                    <div class="sticky doc-sidebar">

                    <br/ >
                    <br/ >
                    <br/ >
                    <br/ >

                    <h4 class="toc-title">Table of Contents</ h4>
                      <nav id="toc"   class=" our-toc "></ nav>
                      <br/ >
                      <br/ >
                   </ div>
                 </ div>

Now, let's find out how to do that cool auto-genrating TOC.

TOC Gen

The sidebar TOC is generated automatically using the TocBot plugin.

First we add the our-toc class to our sidebar nav.Than we wrap our content in a div class called dsquared-content.

Init Tocbot

Now we initialize TocBot with our settings in init.js:


                tocbot.init({

                        tocSelector: '.our-toc',

                        contentSelector: '.dsquared-content',

                       headingSelector: 'h1, h2, h3,h4,h5,h6',

                       collapseDepth: 6,
                 });

Collapse depth refers to if you want it collapsed and to what level with 0 being collapse all and 6 collapse none.

One last step, go through and an id to all your toc headers as that nice highlight indicator will not work if its not set that way.

One last thing, see how my item margins are all nice and aligned? Sometimes in docs we do not follow the header must have a parent rule, we both do it infact. I did a hack to get the margins and that hack is:


                .toc-list .toc-list .toc-list {
                  padding-left: 0px;
                }

Sometime later I will expand it work with more variations of that use case and update this theme.

Visualize File Trees

For API documeents we can use JSTree plugin to show a visual of files such as you viewed earlier. You can embedded html or create a json to describe the structure to visualize. Let me show how to embedd the html.

First, it needs initialized with the name of our embedded html container:


                $('#dsquared-files').jstree();

Than the embedded html right from the example you are reading:


                <div id="dsquared-files">
                     <ul class="list-unstyled">
                        <li data-jstree='{ "selected" : false, "opened" : true }'>In Layouts
                         <ul class="list-unstyled">
                            <li>index.html</ li>
                            <li>favicons and favicon configs</li>
                            <li data-jstree='{ "selected" : false, "opened" : true }'>assets
                               <ul class="list-unstyled">
                                 <li>css</ li>
                                 <li>fonts</ li>
                                 <li>images</ li>
                                 <li>js</ li>
                               </ ul>
                            </ li>
                          </ ul>
                       </ li>
                    </ ul>
               </ div>

And the visual, you will notice that for BS4 I have to set list as unstyled.

Code Highlighting

The display and highlighting of code is accomplished using the prism plugin. Unlike other plugins, once the css files and js files are imported in the html head section the plugin is than initialized automatically.

Using pre and code and some added prism css classes we can display code a nice visual way such as:


                 <pre > <code class="language-javascript line-numbers"  >
                  Stickyfill.add($('.sticky'));
                 </ code></ pre>

Becomes...


                Stickyfill.add($('.sticky'));
